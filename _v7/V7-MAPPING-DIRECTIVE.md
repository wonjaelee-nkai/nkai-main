#### 2.5.2 ⭐ 여행사 일정표 첨부 분석 (신규 기능)

**진입점:** 카이톡 4탭의 **파일 탭** 또는 AI 여행 카드 (`/me/travel`)

**처리 흐름:**
```
사용자 첨부: 여행사 일정표 (PDF / 이미지 / DOCX)
    ↓
1. OCR + Vision API (Gemini or Claude Vision)
2. Sonnet 4.5 일정표 구조 분석:
   - 일자별 일정 추출
   - 호텔·식당·관광지 좌표 매핑
   - 동선 효율성 점수
   - 가격 vs 시장 평균 비교
    ↓
3. 카이가 분석 리포트 생성:
   ✅ 잘 짜인 부분
   ⚠️ 비효율 구간 (예: 동선 X시간 낭비)
   💡 대안 제시 (가성비 더 좋은 호텔, 더 좋은 시간대)
    ↓
4. AI 재생성 일정표 vs 원본 비교 카드
   사용자가 선택: 원본 유지 or AI 버전 사용 or 부분 교체
    ↓
5. 선택 후 → 위 [4단계 원스톱 액션] 카드로 연결
```

**기술 스택:**
- 파일 업로드: Supabase Storage (kaitalk_files 테이블)
- OCR: Google Vision API (이미 연동됨, 5/14 후 활성화 예정 주석 있음)
- 분석: Claude Sonnet 4.5
- 결과 저장: kaitalk_files.analysis_result (JSONB)

#### 2.5.3 ⭐ AI 여행 동시통역 통합 (핵심 차별화)

**기존 양방향 동시통역 = 외국인 게이트** (V7-DIRECTIVE 2.2)
**신규 통합 = 한국인 해외여행자 통역** (이게 더 큰 시장)

**위치:** AI 여행 일정표 자동 생성 후, **여행 중 카이톡에서 자동 활성화**

**작동 방식:**
```
[여행 출발 D-1]
카이톡 푸시: "내일 출발이에요! 동시통역 미리 다운로드?"
사용자 → 동의 → Sonnet 4.5 다국어 모델 사전 로드

[여행 중 - 호텔 체크인]
사용자: 카이톡 마이크 버튼 눌러 한국어 말함
카이: 5개국+ 중 자동 감지 (이미 위치 기반 = 일본이면 일본어)
   → STT (Google) → Sonnet 4.5 번역 → TTS (Google)
호텔 직원: 일본어 응답
카이: → STT → Sonnet 4.5 한국어 번역 → 사용자 화면 표시

[여행 중 - 메뉴판 촬영]
사용자: 메뉴판 사진 촬영
카이: Vision API → 메뉴 텍스트 추출 → Sonnet 4.5 한국어 번역
   + MCC 분석 → 96 좌표 음식 도메인 자동 갱신

[여행 중 - 택시·길찾기]
사용자: "츠키지 시장 가자고 말해줘"
카이: 일본어 음성 출력 (TTS) → 사용자가 택시기사에게 들려줌
카이: GPS 추적 → "거의 다 왔어요" 안내

[여행 중 - 결제 도움]
카이: 환율 자동 계산 (한화 ↔ 현지통화)
카이: 팁 문화 안내 (현지 관습)
카이: 영수증 촬영 → 한국어 번역 + 96 좌표 갱신
```
